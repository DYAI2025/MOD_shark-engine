## Context
SharkEngine has a Docker-based Minecraft test server.
- Dockerfile at project root
- Container name: `sharkengine-server`
- Port mapping: host 25566 → container 25565
- Server properties at `server/server.properties`

## Your Task

Rebuild and restart the Docker Minecraft test server with the latest mod build.

### Steps

1. Check if container `sharkengine-server` is running: `docker ps -a --filter name=sharkengine-server`
2. If running, stop and remove: `docker stop sharkengine-server && docker rm sharkengine-server`
3. Build new image: `docker build -t sharkengine-server .` (from project root)
4. Start container: `docker run -d --name sharkengine-server -p 25566:25565 sharkengine-server`
5. Wait 5 seconds, then check logs: `docker logs sharkengine-server 2>&1 | tail -10`
6. Verify server started successfully (look for "Done" in logs)
7. Report: server address `localhost:25566`, mod version loaded

### Guardrails
- Always stop the old container before starting a new one
- Port 25565 on host may be occupied — use 25566 mapping
- Wait for "Done" message in logs before reporting success
- If build fails, show the error and don't start the container
