## Context
After fixing bugs in SharkEngine, the full test cycle is: build → deploy to Prism → restart Minecraft → test in-game.

## Your Task

Execute the complete build-deploy-test cycle and provide a test checklist.

### Steps

1. Run `./gradlew --no-daemon build` in `sharkengine/` — verify all tests pass
2. Verify JAR contents with `jar tf` (check for tags/block/, lang/de_de.json, sounds/)
3. Copy JAR to Prism Launcher mods folder (see /mod-deploy for path)
4. Re-enable any `.disabled` mods (fabric-api, controlify, yacl)
5. Generate test checklist based on recent git changes:
   - Run `git log --oneline -5` and `git diff HEAD~1 --stat` to see what changed
   - For each changed area, add specific in-game test steps
6. Present the test checklist to the user

### Standard Test Checklist
- **Tutorial:** Place steering wheel → popup appears with readable text + dark panel
- **Localization:** Switch to Deutsch in Options → all texts in German
- **Builder:** Place 4 blocks around steering wheel + thruster → no "invalid attachments"
- **Assembly:** Click "Assemble & Launch" → ship spawns
- **HUD:** Speed, fuel, height show real values (not 0.0)
- **Dismount:** Press Shift → player exits ship
- **Engine-out:** Let fuel run out → ship falls gently, floats on water
- **Sound:** Thruster sound plays during flight

### Guardrails
- Always build before deploying — never copy a stale JAR
- Remind user to FULLY restart Minecraft (not just leave world)
- If build fails, investigate and fix before deploying
