# REQ-F-keyboard-controller-parity: Keyboard and Gamepad Produce Identical Movement

**Type**: Functional

**Status**: Approved

**Priority**: Must-have

**Source**: [US-no-drift-at-rest](../user-stories/US-no-drift-at-rest.md)

**Source stakeholder**: [STK-player](../stakeholders.md)

## Description

The same logical input values (e.g., `moveForward = 1.0, moveStrafe = 0, moveVertical = 0`) must produce identical movement responses in the `HovercraftController` regardless of whether the input was generated by keyboard or gamepad. Input normalization must happen before the controller receives the values.

## Acceptance Criteria

- Given `moveForward = 1.0` sent from keyboard input handler, and `moveForward = 1.0` sent from gamepad input handler, when the `HovercraftController` processes both, then the resulting velocity vector is identical
- Given the parity test (Test I from analysis document), when run against the real controller, then keyboard and gamepad inputs with equal logical values produce movement vectors that differ by < 0.001
